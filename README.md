# 🚀 AI-Powered Insurance for Gig Workers

An AI-based parametric insurance platform designed to protect delivery workers from income loss due to external disruptions like weather, pollution, and social events.

## 📌 Problem Statement

Delivery workers (Zomato, Swiggy, Amazon, etc.) face income loss due to:
- Bad weather (rain, heat, floods)
- Pollution
- Strikes and curfews

Currently, there is **no system to protect their income**, and workers bear all the loss.


## 💡 Solution

We built an **AI-powered parametric insurance system** that:
- Detects real-world disruptions using APIs
- Automatically triggers claims
- Provides instant payouts
- Uses weekly pricing suited for gig workers


## ⚙️ Features

- ✅ AI-based risk assessment
- ✅ Dynamic weekly premium calculation
- ✅ Automatic claim triggering (parametric model)
- ✅ Fraud detection system
- ✅ Instant payout integration
- ✅ Worker & Admin dashboards

---

## 🏗️ System Architecture

1. **Frontend**
   - User onboarding
   - Dashboard (Worker & Admin)

2. **Backend**
   - Policy management
   - Claims engine
   - AI risk calculation

3. **APIs**
   - Weather API (rain, heat detection)
   - Traffic API
   - Location data

4. **Payment System**
   - Razorpay / Stripe / UPI

5. **AI Layer**
   - Risk scoring
   - Fraud detection


## 🔄 Workflow

1. User registers (delivery worker)
2. System assigns risk score using AI
3. Weekly premium is calculated
4. External event occurs (rain, strike, etc.)
5. API detects event
6. Claim is automatically triggered
7. System verifies data (fraud detection)
8. Instant payout is processed


## 🛠️ Tech Stack

- Frontend: React.js, Tailwind CSS  
- Backend: Node.js, Express.js  
- Database: MongoDB  
- AI/ML: Python, TensorFlow  
- APIs: Weather API, Google Maps API  
- Payments: Razorpay / Stripe / UPI  
- Cloud: AWS
